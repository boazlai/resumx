# Feature Registry

This directory tracks the current state of the user-facing product, one feature per Markdown file.

## How To Use This Directory

- Before changing a user-facing feature, read the corresponding feature file here.
- After changing a feature, update its status, current behavior, missing pieces, and next steps.
- If you touch a new user-facing feature and no file exists yet, create one in this directory.
